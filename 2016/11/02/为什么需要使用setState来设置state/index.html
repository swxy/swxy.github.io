为什么需要使用setState来设置state

By swxy

Published 2016-11-02

看React官网文档时,提及一段话

Do Not Modify State Directly
For example, this will not re-render a component:

// Wrong
this.state.comment = ‘Hello’;

Instead, use setState():

// Correct
this.setState({comment: ‘Hello’});

这里只是提及了一下编程过程如何设置state，然而并没有给出解释：

  1. 为什么使用this.state.comment来设置state中的comment就不会重新渲染一个组件，而使用setState就能？
  2. 如果使用this.state.comment来设置值，能否手动更新组件？
  3. 使用setState设置值时，能否不让组件更新？

首先我们需要明确的一点React不是一个MVC框架，而主要注重于MVC中的V(view)层。具体可以看官网博客《Why did we build React?》。
可以简单的理解为React做的事情就是根据数据来自动渲染页面，类似于一个前端模板库，但提供使用JSX快速开发，事件处理，组件开发，生命周期等等功能。
